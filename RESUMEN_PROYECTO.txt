FRAMEWORK: Django 6.0.2
BASE DE DATOS: SQLite3
UBICACIÓN: c:\Users\ccih\Documents\Django\Proyectos\Api-rest

DESCRIPCIÓN GENERAL:
Este proyecto es una API REST completa para la gestión de películas que permite
a los usuarios registrarse, autenticarse, explorar un catálogo de películas,
filtrarlas por diferentes criterios, calificarlas, marcarlas como favoritas
y gestionar su lista personal de películas vistas y por ver.

5. MODELOS DE DATOS (BASE DE DATOS)
================================================================================

-------------------------------------------
5.1 APP AUTHENTICATION - Modelo CustomUser
-------------------------------------------

DESCRIPCIÓN: Usuario personalizado que extiende AbstractUser de Django

CAMPOS:
- email (EmailField): Correo electrónico único (usado para login)
- username (CharField): Nombre de usuario único
- password (CharField): Contraseña hasheada
- avatar (ImageField): Imagen de perfil del usuario (opcional)
- Hereda campos de AbstractUser: first_name, last_name, date_joined, etc.

CARACTERÍSTICAS ESPECIALES:
- USERNAME_FIELD: 'email' (login con email en lugar de username)
- REQUIRED_FIELDS: ['username', 'password']
- Avatares guardados en: media/avatars/{user_id}/

-------------------------------------------
5.2 APP FILMS - Modelo Film
-------------------------------------------

DESCRIPCIÓN: Representa una película en el catálogo

CAMPOS:
- id (UUIDField): Identificador único UUID (clave primaria)
- title (CharField): Título de la película (max 150 caracteres)
- year (PositiveIntegerField): Año de estreno (default: 2000)
- review_short (TextField): Reseña corta (opcional)
- review_large (TextField): Reseña larga (opcional)
- trailer_url (URLField): URL del trailer en YouTube/etc (opcional)
- genres (ManyToManyField): Relación muchos-a-muchos con FilmGenre
- image_thumbnail (ImageField): Miniatura de la película (opcional)
- image_wallpaper (ImageField): Wallpaper de la película (opcional)
- favorites (IntegerField): Contador de favoritos (calculado automáticamente)
- average_note (FloatField): Nota media de 0-10 (calculada automáticamente)

MÉTODO ESPECIAL:
- path_to_film(): Define la ruta de subida de imágenes (films/{uuid}/)
- update_film_stats(): Señal que actualiza stats cuando cambia FilmUser
- __str__(): Retorna "Título (Año)"

ESTADÍSTICAS AUTOMÁTICAS:
- El campo 'favorites' se actualiza automáticamente contando los favoritos
- El campo 'average_note' calcula la media de todas las notas recibidas
- Ambos se actualizan mediante señales Django (post_save)

META:
- verbose_name: 'Película'
- verbose_name_plural: 'Películas'
- ordering: ['title']

-------------------------------------------
5.3 APP FILMS - Modelo FilmGenre
-------------------------------------------

DESCRIPCIÓN: Géneros cinematográficos (Acción, Drama, Comedia, etc)

CAMPOS:
- id (AutoField): ID automático
- name (CharField): Nombre del género (único, max 50 caracteres)
- slug (SlugField): Slug único para URLs amigables

MÉTODOS:
- save(): Genera el slug automáticamente desde el nombre
- __str__(): Retorna el nombre del género

META:
- verbose_name: 'Género de película'
- verbose_name_plural: 'Géneros de película'
- ordering: ['name']

-------------------------------------------
5.4 APP FILMS - Modelo FilmUser
-------------------------------------------

DESCRIPCIÓN: Relación entre usuarios y películas (lista personal)

CAMPOS:
- user (ForeignKey): Referencia al usuario
- film (ForeignKey): Referencia a la película
- state (PositiveSmallIntegerField): Estado de visualización
  * 0: Sin estado
  * 1: Vista
  * 2: Quiero ver
- favorite (BooleanField): Si es favorita del usuario
- note (PositiveSmallIntegerField): Calificación de 0-10 (opcional)
- review (TextField): Reseña personal del usuario (opcional)

META:
- unique_together: ('user', 'film') - Un usuario solo puede tener una relación por película
- ordering: ['film__title']

SEÑAL POST_SAVE:
Cuando se guarda un objeto FilmUser, se dispara Film.update_film_stats que:
1. Cuenta todos los favoritos de esa película
2. Calcula la nota media de todas las calificaciones
3. Actualiza automáticamente el objeto Film

7. VISTAS Y ENDPOINTS DE LA API
================================================================================

-------------------------------------------
7.1 AUTHENTICATION APP - Endpoints
-------------------------------------------

BASE URL: /api/auth/

1. POST /api/auth/login/
   Vista: LoginView (APIView)
   Descripción: Autentica usuario con email y password
   Body: { "email": "user@example.com", "password": "password123" }
   Respuesta: Datos del usuario serializado + sesión iniciada
   Status: 200 OK | 401 UNAUTHORIZED

2. POST /api/auth/logout/
   Vista: LogoutView (APIView)
   Descripción: Cierra la sesión del usuario actual
   Requiere: Autenticación
   Status: 200 OK

3. POST /api/auth/signup/
   Vista: SignupView (CreateAPIView)
   Descripción: Registro de nuevos usuarios
   Body: { "email", "username", "password", "avatar"? }
   Respuesta: Datos del usuario creado
   Status: 201 CREATED

4. POST /api/auth/reset/
   Vista: django_rest_passwordreset
   Descripción: Recuperación de contraseña por email
   Incluye: Múltiples endpoints para reset de password

5. GET /api/user/profile/
   PATCH /api/user/profile/
   Vista: ProfileView (RetrieveUpdateAPIView)
   Descripción: Ver y actualizar perfil del usuario autenticado
   Requiere: Autenticación
   Métodos: GET, PATCH (no PUT)

-------------------------------------------
7.2 FILMS APP - Endpoints
-------------------------------------------

BASE URL: /api/

1. GET /api/films/
   Vista: FilmViewSet (ReadOnlyModelViewSet)
   Descripción: Lista todas las películas con paginación
   Paginación: 8 películas por página
   
   FILTROS DISPONIBLES:
   - Búsqueda: ?search=termino
     * Busca en: title, year, genres__name
   - Ordenamiento: ?ordering=campo
     * Campos: title, year, genres__name, favorites, average_note
     * Descendente: -campo
   - Filtros avanzados:
     * ?year__gte=2010 (películas desde 2010)
     * ?year__lte=2020 (películas hasta 2020)
     * ?genres={genre_id} (por género específico)
   
   RESPUESTA PAGINADA:
   {
     "count": 100,              # Total de películas
     "num_pages": 13,           # Número de páginas
     "page_number": 1,          # Página actual
     "page_size": 8,            # Películas por página
     "next_link": "page=2",     # Link a siguiente página
     "previous_link": null,     # Link a página anterior
     "results": [...]           # Array de películas
   }

2. GET /api/films/{id}/
   Vista: FilmViewSet
   Descripción: Detalle de una película específica por UUID
   Respuesta: Objeto Film completo con géneros anidados

3. GET /api/genres/
   Vista: GenreViewSet (ReadOnlyModelViewSet)
   Descripción: Lista todos los géneros cinematográficos
   Respuesta: Array de géneros con sus películas

4. GET /api/genres/{slug}/
   Vista: GenreViewSet
   Descripción: Detalle de un género por slug
   lookup_field: 'slug' (usa slug en lugar de id)
   Ejemplo: /api/genres/accion/
   Respuesta: Género con lista de películas de ese género

5. GET /api/userfilms/
   Vista: FilmUserViewSet (APIView)
   Descripción: Obtiene lista personal de películas del usuario
   Requiere: Autenticación (SessionAuthentication)
   Permisos: IsAuthenticated
   Respuesta: Array de FilmUser con películas completas

6. POST /api/userfilms/
   Vista: FilmUserViewSet
   Descripción: Crear/actualizar relación usuario-película
   Requiere: Autenticación
   Body: {
     "uuid": "film-uuid",
     "favorite": true/false,
     "state": 0|1|2,
     "note": 0-10,
     "review": "texto"
   }
   COMPORTAMIENTO ESPECIAL:
   - Si state=0, elimina la relación (película sin estado)
   - Usa update_or_create para evitar duplicados
   Status: 200 OK | 404 NOT_FOUND

8. CARACTERÍSTICAS ESPECIALES
================================================================================

-------------------------------------------
8.1 SISTEMA DE PAGINACIÓN PERSONALIZADA
-------------------------------------------

CLASE: ExtendedPagination
- Hereda de PageNumberPagination
- page_size: 8 elementos por página
- Respuesta personalizada con metadata completa:
  * count: Total de elementos
  * num_pages: Número total de páginas
  * page_number: Página actual
  * page_size: Tamaño de página
  * next_link: Solo los parámetros de query (no URL completa)
  * previous_link: Solo los parámetros de query
  * results: Array de resultados

-------------------------------------------
8.2 SISTEMA DE FILTROS AVANZADOS
-------------------------------------------

BACKEND: DjangoFilterBackend + SearchFilter + OrderingFilter

FILTROS EN FILMS:
1. Búsqueda de texto: ?search=palabra
   - Campos: title, year, genres__name
2. Ordenamiento: ?ordering=campo
   - Campos: title, year, genres__name, favorites, average_note
3. Filtros de rango: 
   - ?year__gte=2010
   - ?year__lte=2020
4. Filtro exacto:
   - ?genres=id_del_genero

COMBINACIÓN:
Se pueden combinar múltiples filtros:
/api/films/?year__gte=2020&ordering=-average_note&search=accion

-------------------------------------------
8.3 ACTUALIZACIÓN AUTOMÁTICA DE ESTADÍSTICAS
-------------------------------------------

MECANISMO: Señales Django (post_save)

CUANDO UN USUARIO:
- Marca/desmarca como favorita
- Califica una película
- Cambia el estado de visualización

SE EJECUTA AUTOMÁTICAMENTE:
Film.update_film_stats() que:
1. Cuenta favoritos: FilmUser.objects.filter(film=X, favorite=True).count()
2. Calcula nota media: Sum(note) / Count(note)
3. Actualiza Film.favorites y Film.average_note
4. Guarda el objeto Film

CONEXIÓN DE SEÑAL:
post_save.connect(Film.update_film_stats, sender=FilmUser)

9. FLUJO DE TRABAJO DEL SISTEMA
================================================================================

-------------------------------------------
9.1 REGISTRO E INICIO DE SESIÓN
-------------------------------------------

REGISTRO:
1. Usuario envía POST a /api/auth/signup/
2. Se valida email y username únicos
3. Se hashea la contraseña
4. Se crea CustomUser
5. Se retorna datos del usuario

LOGIN:
1. Usuario envía POST a /api/auth/login/ con email y password
2. Django autentica con authenticate()
3. Si es válido, ejecuta login()
4. Crea sesión en cookies
5. Retorna datos del usuario serializado

-------------------------------------------
9.2 EXPLORACIÓN DE PELÍCULAS
-------------------------------------------

LISTADO:
1. Cliente hace GET a /api/films/
2. Puede aplicar filtros y ordenamiento
3. API retorna 8 películas paginadas
4. Cliente usa next_link/previous_link para navegar

DETALLE:
1. Cliente hace GET a /api/films/{uuid}/
2. API retorna película completa con géneros

BÚSQUEDA:
1. Cliente hace GET a /api/films/?search=termino
2. API busca en title, year, y genres__name
3. Retorna resultados paginados

-------------------------------------------
9.3 GESTIÓN DE LISTA PERSONAL
-------------------------------------------

VER MI LISTA:
1. Usuario autenticado hace GET a /api/userfilms/
2. API filtra FilmUser por usuario actual
3. Retorna lista con películas completas

AÑADIR/ACTUALIZAR PELÍCULA:
1. Usuario hace POST a /api/userfilms/ con datos
2. API busca la película por UUID
3. Usa update_or_create para evitar duplicados
4. Actualiza favorite, state, note, review
5. Si state=0, elimina la relación
6. Dispara señal post_save
7. Film.update_film_stats actualiza estadísticas

-------------------------------------------
9.4 ACTUALIZACIÓN DE PERFIL
-------------------------------------------

1. Usuario hace PATCH a /api/user/profile/
2. Puede actualizar username, password, avatar
3. Email NO puede modificarse (se elimina del validated_data)
4. Username y email se validan por unicidad
5. Password se hashea automáticamente
6. Se retorna usuario actualizado

10. BASE DE DATOS - RELACIONES
================================================================================

DIAGRAMA DE RELACIONES:

CustomUser (1) ────(M)─── FilmUser ───(M)────(1) Film
                                                  │
                                                  │ (M)
                                                  │
                                                  └────(M) FilmGenre

EXPLICACIÓN:
- Un CustomUser puede tener muchos FilmUser (su lista personal)
- Una Film puede tener muchos FilmUser (vista por muchos usuarios)
- Una Film puede tener muchos FilmGenre (varios géneros)
- Un FilmGenre puede tener muchas Film (muchas películas del género)

CASCADAS:
- FilmUser.user → CASCADE: Si se borra usuario, se borran sus FilmUser
- FilmUser.film → CASCADE: Si se borra película, se borran sus FilmUser
